How It Works

ML Pipeline

Six-phase process from raw clinical data to real-time web prediction.

1

Data Loading & EDA

Loaded cleaned_dataset_Thyroid1.csv, inspected shape, nulls, duplicates, class distribution.

2

Feature Engineering

Dropped 11 low-value / measured-flag columns. Retained 15 clinically relevant biomarkers.

3

Correlation Heatmap

Seaborn coolwarm heatmap to identify multicollinearity and guide feature selection.

4

Train / Test Split

80/20 stratified split with random_state=42 for reproducibility.

5

Model Training

Random Forest Classifier (primary) + KNN (comparison). Evaluated with accuracy, confusion matrix, classification report.

6

Serialization & Deployment

Best model saved as thyroid_model.pkl via pickle, served through Flask REST endpoints.

Development Journey

Project Phases

📁
Phase 1

Data Acquisition & Cleaning

Loaded cleaned_dataset_Thyroid1.csv, checked for nulls, removed duplicates, explored the sex column and class distribution of binaryClass.

📊
Phase 2

Exploratory Data Analysis

Generated correlation heatmap (20×16 figure, coolwarm palette). Identified low-correlation and redundant measured-flag columns for removal.

✂️
Phase 3

Feature Selection

Dropped 11 columns: TSH/T3/TT4/T4U/FTI measured flags, query columns, lithium, hypopituitary, psych. Retained 15 meaningful features.

🤖
Phase 4

Model Training & Evaluation

Trained RandomForestClassifier on 80% data. Evaluated with accuracy score, confusion matrix heatmap, and full classification report.

📐
Phase 5

KNN Comparison

Trained KNeighborsClassifier on same split for comparison. Random Forest outperformed on thyroid dataset.

🌐
Phase 6

Flask Deployment

Model serialized with pickle, served via Flask. 4-page web app: Login → Dashboard → History → About with full animations.
